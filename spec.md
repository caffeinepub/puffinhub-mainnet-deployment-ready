- **Principal Context**: Unified principal context shared across the entire application, ensuring consistent identity detection for all features including profile management and game scores
- **Persistent Session Management**: Maintain user authentication state across page navigation with consistent principal access throughout the application
- **Logout Functionality**: Allow users to securely log out and clear the session state

## Connected Wallet Integration
- **Plug Wallet Exclusive**: Users connect exclusively through Plug Wallet for all ICP transactions and balance viewing
- **Wallet Connection Interface**: Dedicated wallet connection section allowing users to connect their Plug wallet
- **Connected Wallet Display**: Show connected Plug wallet information including connection status with puffin-themed styling
- **Enhanced ICP Ledger Balance Fetching**: Connected Plug wallet fetches live ICP balance using direct ICP ledger integration when the user actively clicks a "Check ICP Balance" button. The system uses `@dfinity/agent` with `Actor` and `HttpAgent` imports, connects to the ICP ledger canister (`ryjl3-tyaaa-aaaaa-aaaba-cai`), converts the user's principal to Account ID using `principalToAccountIdentifier` from the account ID utility, creates a ledger actor with Plug's agent, calls `ledger.account_balance({account: accountId})`, and converts the returned e8s amount to human-readable ICP units for accurate real-time balance display in the Connected Wallet section
- **Direct Ledger Integration**: Connected Plug wallet communicates directly with the ICP ledger canister (`ryjl3-tyaaa-aaaaa-aaaba-cai`) for transaction processing and balance queries
- **Frontend-Only Transactions**: All ICP transfers occur through Plug wallet API without requiring backend canister involvement
- **Icon-Free Wallet Display**: Wallet section displays text-only interface without wallet logos, token icons, or brand imagery while maintaining puffin-themed styling
